Trajectory inference, aka pseudotime

Trajectory inference assumes that cells are sampled during various stages of a transition from a cell type or state to another type or state. By identifying trajectories that connect cells based on similarilty in gene expression, one can gain insights into lineage relationships and developmental trajectories. To quantitate the measure of biological progress outside of defined time-points we use a metric called pseudotime, which is defined as a distance metric between the starting cell and ending cell along the trajectory.

Analysis of discrete clusters can hide interesting continous behaviors in cell populations. By ordering cells based on an expression trajectory one can uncover novel patterns of gene expression.

There are many methods

knitr::include_graphics("img/dynverse.webp")

Check out the dynverse for help with algorithm selection. They benchmarked > 60 methods, and offer some tools to run multiple algorithms on the same data.

Simplest method (PCA)

In some datasets, particularly developmental datasets it is often the case that a principal component may separate out cells based on known developmental time. This can be used a pseudotime metric, which is simply the PC1 scores for each cell. As an example, scRNA-seq analyses of liver development and Beta-cell maturation used PC1 to examine pseudotime.

Destiny

Non-linear dynamics are not always captured by PCA. Diffusion maps offer an alternative, non-linear transformation approach that infers the temporal order of differentiating cells by modeling it as a diffusion process. This can better resolve complex trajectories and branching than linear methods like PCA. Well use the R package destiny to look at diffusion maps and diffusion pseudotime.

# Generating a diffusion map requires a matrix of expression data:
# prepare a counts matrix
logcounts <- GetAssayData(sobj, "data")

# transpose matrix (genes as columns, cells as rows)
input_matrix <- t(logcounts[VariableFeatures(sobj), ])

# generate a diffusion map
set.seed(42)
dm <- DiffusionMap(as.matrix(input_matrix))

# store the diffusion map as a custom dimensional reduction in the Seurat object
sobj[["DM"]] <- CreateDimReducObject(embeddings = dm@eigenvectors, key = "DM_", assay = DefaultAssay(sobj))

# plot the diffusion map
DimPlot(sobj, reduction = "DM", group.by = "cell_type1") + ggtitle("Diffusion Map")

Next we calculate diffusion pseudotime using DPT.

# calculate the diffusion pseudotime (DPT)
dpt <- DPT(dm)

# color single cells on diffusion map plot according to DPT
p1 <- plot(dpt, 1:2) + ggtitle("Diffusion Pseudotime (DPT)")

# create data.frame for easy plotting of DPT
tmp <- data.frame(DC1 = dm$DC1,
                  DC2 = dm$DC2,
                  timepoint = cell_type,
                  dpt = dpt$DPT1)

# color single cells on diffusion map plot according to timepoint
p2 <- ggplot(tmp, aes(x = DC1, y = DC2, colour = timepoint)) +
    geom_point() +
    theme_classic() +
    ggtitle("Timepoint")

plot_grid(p1, p2, align = "hv", ncol = 2, rel_widths = c(1, 1))


# plot timepoint vs. DPT
ggplot(tmp, aes(timepoint, dpt, colour = timepoint)) +
  geom_point() + 
  geom_jitter() +
  ggtitle ("Timepoint vs. DPT")

PCA and diffusion maps can be good for simple trajectories or pathway activation systems e.g.T-cell activation, drug pertubation, single cell type responses.

Slingshot

For more complex trajectories, simply using a single PC or a diffusion componenet may not be sufficient. Systems with bi or tri-furcating trajectories wont be well fit within a single dimension.

For this next analysis we will use a dataset taken from a single cell RNA-seq study of hepatocyte development.

sobj <- readRDS("obj/slingshot.rds")

Identifying differentially expressed genes along a trajectory

We can use a GAM (or other models) to compare pseudotimes to gene expression values to identify genes that vary significantly with pseudotime. Using a GAM can uncover non-linear expression patterns (i.e.not just up or down). See this blog for a nice description.

# select the ptime values 
ptime <- sce$slingPseudotime_2

# get cells in that lineage
lineage_cells <- colnames(sce)[!is.na(ptime)]

# remove values for cells not in the lineage
ptime <- ptime[!is.na(ptime)]

# just test variable genes to save some time
genes_to_test <- VariableFeatures(sobj)[1:1000]

# get log normalized data to test
cnts <- logcounts(sce)[genes_to_test, lineage_cells]

# fit a GAM with a loess term for pseudotime
gam.pval <- apply(cnts, 1, function(z){
    d <- data.frame(z = z, ptime = ptime)
    tmp <- suppressWarnings(gam(z ~ lo(ptime), data=d))
    p <- summary(tmp)[4][[1]][1, 5]
    p
})

# adjust pvalues 
res <- tibble(
  id = names(gam.pval),
  pvals = gam.pval,
  qval = p.adjust(gam.pval, method = "fdr")) %>% 
  arrange(qval)

head(res)

tradeSeq

tradeSeq is a package that allows analysis of gene expression along trajectories (such as those found with Slingshot). For every gene in the dataset, tradeSeq fits a GAM (as explained in the previous section). It then allows statistical inference on the GAM by assessing contrasts of the parameters of the fitted GAM, aiding in interpreting complex datasets. For more details, see the tradeSeq paper. The following is from the tradeSeq tutorial.

Lets use tradeSeq to look at our Slingshot data set from the previous section:

# tradeSeq needs a counts matrix and lineage curves as minimal input
crv <- sds
counts <- as.matrix(assays(sce)$counts)
clusters <- sce$seurat_clusters

The following two code blocks we will NOT run, as they both take several minutes. These two code blocks are fitting the tradeSeq negative binomial generalized additive model (NB-GAM). For an in-depth description of whats going on, see this vignette. Basically, we are using the NB-GAM framework in tradeSeq to smooth each genes expression in each lineage. Smoothers can be decomposed into a set of basis functions, which are joined together at knot points (simply called knots).

We run the evaluateK function in order to determine the correct number of knots. This returns the AIC (Akaike Informaction Criterion) for each gene fitted with each number of knots defined in the function (typically the interesting range is 3 to 10 knots) in a series of diagnostic plots.

set.seed(5)
icMat <- evaluateK(counts = counts, sds = crv, k = 3:10, nGenes = 200, verbose = T)

The middles panels show that the drop in AIC levels off if the number of knots is increased beyond 6, so well choose that number of knots to fit the tradeSeq models using the fitGAM function. nknots = 6 is also the default for the fitGAM function:

set.seed(7)
sce <- fitGAM(counts = counts, sds = sds, nknots = 6, genes = 1:1000, verbose = TRUE, sce=TRUE)

# Load in the SingleCellExperiment created by running `fitGAM` above
sce <- readRDS("obj/tradeSeq.rds")

# plot our Slingshot lineage trajectories, this time illustrating the new tradeSeq knots
plotGeneCount(curve = sds, counts = counts,
              clusters = clusters,
              models = sce)

With tradeSeq, we can examine differential gene expression in a variety of ways. We can check if a gene is associated with a particular lineage using associationTest, discover progenitor marker genes using startVsEndTest, compare specific pseudotime values with a lineage, discover differentiated cell type markers using diffEndTest, discover genes with different expression patterns using patternTest, and identify early drivers of differentiation using earlyDETest.

PAGA

PAGA (partition-based graph abstraction) is a python module available in the python counterpart to seurat known as scanpy. PAGA provides both a very nice visualization technique, and a method for trajectory analysis. Whereas PC and DPT analyses are good for simple trajectories, and Slingshot is good for bi- and multifuracting trajectories, PAGA is good for more complex trajectories.

RNA velocity

Based on a cool concept that while the exonic transcriptome represents the current state of a cell, the intronic transcriptome represents what the cell is going to be in the near future. By introducing a transcriptional dynamic model, RNA velocity analysis predicts the directional flow of cell state transition, which greatly expands the application of scRNA-seq to capture dynamics of molecular changes. There is a python and R package implementation for performing analysis, both of which requires processing a bam file to quantitate unspliced and spliced mRNAs per cell (which we cant do within the constraints of the workshop). Recently a scalable (python) toolkit for RNA velocity analysis in single cells called scVelo was published, which can do additional analysis such as estimating the directional velocity pseudotime.

http://velocyto.org/ https://github.com/velocyto-team/velocyto.R https://scvelo.readthedocs.io/

Recommendations:

  1. Limit analysis to relevant cell populations (i.e.dont try to infer trajectories where you know they dont exist)

  2. Start with simple methods and trajectories.

  3. Complement pseudotime trajectories with RNA velocity analyses to provide directionality and orthogonal validation.
